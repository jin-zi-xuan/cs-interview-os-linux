# cs-interview-os-linux

## 1. 进程、线程、协程有什么区别？为什么大厂高并发喜欢用协程（如 Go）？

### 💡 通俗理解与底层逻辑
* **进程（大工厂）：** 系统分配资源（如内存）的最小单位。相互隔离，创建和销毁的开销极大。
* **线程（流水线工人）：** CPU 调度的最小单位。多个线程跑在一个进程里，共享工厂的资源。但线程的调度归“操作系统内核”管，切换时需要陷入内核态，开销中等（微秒级）。
* **协程（工人的极速分身）：** 用户态的轻量级线程。它的调度完全由程序员写的代码（如 Go 的 runtime）控制，**不经过操作系统内核**。切换只是在用户态修改几个寄存器，开销极小（纳秒级），且内存占用低（通常几 KB）。

### 📝 面试满分回答模板
**Q：请简述进程、线程、协程的区别？**
**A：** 它们的核心区别体现在资源管理、调度者和切换开销上：
1. **定义与资源：** 进程是操作系统分配资源的最小单位，拥有独立的虚拟内存空间；线程是 CPU 调度的最小单位，属于同一个进程的线程共享进程的内存和资源；协程是运行在单个线程中的“用户态轻量级线程”。
2. **切换开销（核心考点）：** - 进程切换开销最大，需要切换虚拟内存地址空间（页表）、寄存器等，并伴随内核态的切换。
   - 线程切换不需要切换地址空间，但依然需要陷入内核态来保存和恢复 CPU 的栈和寄存器状态。
   - 协程的切换完全在用户态完成，无需操作系统的系统调用干预，仅需保存极少的寄存器状态，开销极低。
3. **并发能力：** 一台普通服务器最多支撑数千个活跃线程，而由于协程的轻量化（如 Goroutine 初始栈仅 2KB），一台机器可以轻松并发数万甚至数十万个协程，极大地提升了 I/O 密集型应用的并发吞吐量。

## 2. 什么是死锁？产生死锁的四个必要条件是什么？如何预防？

### 💡 通俗理解与底层逻辑
* **死锁的本质：** 两个或多个进程在争夺资源时，互相等待对方释放资源，导致所有人都卡住无法继续执行的僵局。（就像你抱着键盘等鼠标，我抱着鼠标等键盘）。
* **四大天王（必要条件）：**
  1. **互斥：** 资源只能同时给一个人用。
  2. **请求与保持：** 自己手里的资源不肯放，还眼巴巴盯着别人手里的。
  3. **不可剥夺：** 不能使用暴力抢夺别人的资源。
  4. **环路等待：** 形成了一个 A 等 B，B 等 C，C 等 A 的闭环。
* **破阵之法（预防策略）：** 破坏任意一个条件即可。在工程中最常用的是**破坏环路等待条件**，即要求所有进程按**统一固定的顺序**去申请资源（比如规定必须先申请锁 A，再申请锁 B），这样就不会出现互相交叉等待的情况。

### 📝 面试满分回答模板
**Q：请简述什么是死锁及产生死锁的四个必要条件？**
**A：** 死锁是指两个或两个以上的进程在执行过程中，因争夺资源而造成的一种互相等待的现象，若无外力作用，它们都将无法推进下去。
产生死锁必须同时满足四个条件：
1. **互斥条件：** 资源在一段时间内只能被一个进程占有。
2. **请求和保持条件：** 进程已经保持了至少一个资源，但又提出了新的资源请求，而该资源已被其他进程占有，此时请求进程阻塞，但对自己已获得的资源保持不放。
3. **不可剥夺条件：** 进程已获得的资源，在未使用完之前，不能被强行剥夺，只能由自己主动释放。
4. **循环等待条件：** 发生死锁时，必然存在一个进程——资源的环形链。

**Q：在写代码时，通常如何预防死锁？**
**A：** 预防死锁的核心在于打破上述四个必要条件之一。在实际编码中：
1. **破坏请求与保持：** 采用一次性分配策略，即进程在运行前一次性申请它所需要的全部资源。
2. **破坏不可剥夺条件：** 使用带有超时机制的锁（例如 Java 的 `ReentrantLock.tryLock(timeout)`）。如果在一段时间内无法获取所需资源，则主动释放已占有的资源。
3. **破坏循环等待条件（最常用）：** 实行**资源有序分配策略**。规定系统中的所有资源都有一个全局的线性顺序，所有进程必须严格按照这个给定的顺序去申请资源，从而避免环路的产生。

## 3. 什么是虚拟内存？为什么要有虚拟内存？

### 💡 通俗理解与底层逻辑
* **核心目的：** 为每个进程提供一个独立的、连续的虚拟地址空间。
* **解决的问题：**
  1. **安全性：** 进程间内存隔离，防止恶意篡改其他进程数据。
  2. **扩展性：** 让程序可以运行在比物理内存更大的地址空间上（通过换入换出技术）。
  3. **效率：** 物理内存可以是非连续的（碎片化），但由于页表映射的存在，进程看到的依然是连续的地址。
* **实现机制：** CPU 里的 **MMU（内存管理单元）** 负责把虚拟地址转换成物理地址。如果地址不在内存中，则触发 **缺页中断 (Page Fault)** 从硬盘读取。

### 📝 面试满分回答模板
**Q：请解释什么是虚拟内存？它的主要作用是什么？**
**A：** 虚拟内存是操作系统提供的一种内存管理技术。它使得每个进程都拥有一套独立的、完整的虚拟地址空间，而不需要关心物理内存的实际布局。
其主要作用包括：
1. **进程隔离：** 每个进程都有自己的页表，互不干扰，保障了系统的稳定性。
2. **内存扩容：** 配合硬盘（Swap/虚拟内存文件），允许运行内存占用超过物理 RAM 大小的程序。
3. **内存共享：** 不同的虚拟地址可以映射到同一块物理内存，实现高效的进程间通信。

**Q：简述分页机制及缺页中断的处理流程。**
**A：** 操作系统将虚拟内存和物理内存划分为固定大小的块（通常为 4KB）。
1. 当进程访问一个虚拟地址时，**MMU** 会通过查**页表**来寻找对应的物理帧。
2. 如果该页在页表中被标记为“无效”（即不在物理内存中），则触发 **缺页中断**。
3. 操作系统捕获中断，从硬盘中加载该页到空闲的物理帧。
4. 如果物理内存已满，则根据页面置换算法（如 **LRU** 算法）剔除一个旧页。
5. 更新页表，重新执行刚才失败的指令。

**Q：什么是 TLB？它起什么作用？（加分项）**
**A：** TLB（Translation Lookaside Buffer）是 MMU 内部的一个高速缓存，专门用来存放最近访问过的页表项。由于访问内存页表较慢，有了 TLB 后，大部分地址转换可以直接在 CPU 硬件内完成，极大提高了寻址效率。
