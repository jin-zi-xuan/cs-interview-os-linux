# cs-interview-os-linux

## 1. 进程、线程、协程有什么区别？为什么大厂高并发喜欢用协程（如 Go）？

### 💡 通俗理解与底层逻辑
* **进程（大工厂）：** 系统分配资源（如内存）的最小单位。相互隔离，创建和销毁的开销极大。
* **线程（流水线工人）：** CPU 调度的最小单位。多个线程跑在一个进程里，共享工厂的资源。但线程的调度归“操作系统内核”管，切换时需要陷入内核态，开销中等（微秒级）。
* **协程（工人的极速分身）：** 用户态的轻量级线程。它的调度完全由程序员写的代码（如 Go 的 runtime）控制，**不经过操作系统内核**。切换只是在用户态修改几个寄存器，开销极小（纳秒级），且内存占用低（通常几 KB）。

### 📝 面试满分回答模板
**Q：请简述进程、线程、协程的区别？**
**A：** 它们的核心区别体现在资源管理、调度者和切换开销上：
1. **定义与资源：** 进程是操作系统分配资源的最小单位，拥有独立的虚拟内存空间；线程是 CPU 调度的最小单位，属于同一个进程的线程共享进程的内存和资源；协程是运行在单个线程中的“用户态轻量级线程”。
2. **切换开销（核心考点）：** - 进程切换开销最大，需要切换虚拟内存地址空间（页表）、寄存器等，并伴随内核态的切换。
   - 线程切换不需要切换地址空间，但依然需要陷入内核态来保存和恢复 CPU 的栈和寄存器状态。
   - 协程的切换完全在用户态完成，无需操作系统的系统调用干预，仅需保存极少的寄存器状态，开销极低。
3. **并发能力：** 一台普通服务器最多支撑数千个活跃线程，而由于协程的轻量化（如 Goroutine 初始栈仅 2KB），一台机器可以轻松并发数万甚至数十万个协程，极大地提升了 I/O 密集型应用的并发吞吐量。
